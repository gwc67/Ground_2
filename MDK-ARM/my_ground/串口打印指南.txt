================================================================================
串口打印指南 —— 怎么把数据打印到 USART2（串口屏那一路）
（面向：用 C，单片机接触不多；只想知道"怎么 printf 到串口2 看到数据"）
================================================================================


================================================================================
零、先搞清楚：串口2 在本工程里叫什么
================================================================================

  本工程的串口不是裸的 huart2 直接用，而是包了一层。每个串口都建了一个
  "基类实例" + 一个全局指针，全工程统一用指针访问：

    USART1 (500000, ANO 飞控协议)    → pstbase_ground_uart
    USART2 (9600,  串口屏这一路)     → pstbase_screen_uart   ← 本指南的主角
    USART3 (500000, IT 收发)         → pstbase_usart3_uart

  所以"打印到串口2" = 用 pstbase_screen_uart 这个指针发数据。
  别去直接调 HAL_UART_Transmit(&huart2,...)，绕过了 TX 队列，会和屏的
  正常收发打架。统一走项目的 uart_printf_v。

  一句话：你要打印到串口2，就是调 uart_printf_v(pstbase_screen_uart, 0, "...", ...)。


================================================================================
一、发送链路（你调一句，底下发生了什么）
================================================================================

  你的代码                    底层
  ──────────────────────────  ──────────────────────────────────────────
  uart_printf_v(...)      →  uart_log.c  : vsnprintf 格式化进 128B buf
                              uart_base.c : uart_transmit() 分发
                              uart_true.c : memcpy 进 TX 队列（非阻塞）
                                             空闲时出队 → DMA 发到 USART2
                              usart.c     : huart2 硬件 TX (PA2, 9600)

  关键性质（来自 uart_log.h 自带说明）：
    · 调用立即返回，不阻塞你（TX 是队列，DMA 在后台发）
    · 你要自己加 \r\n，接口不自动追加
    · 单条最大 127 字节（含 '\0'），超长被截断
    · TX 队列满了会丢帧、不阻塞——所以别在收数回调里高频发


================================================================================
二、最简写法（直接调，照抄能跑）
================================================================================

  需要两个头文件（incList 里已有 my_ground/Drv_Uart，不用改工程）：

    #include "uarts.h"      // 提供 pstbase_screen_uart
    #include "uart_log.h"   // 提供 uart_printf_v 和宏

  然后：

    void my_print_demo(void)
    {
        int x = 12, y = 34;

        /* 第1参=串口句柄, 第2参=级别(传0), 第3参起=printf格式串+参数 */
        uart_printf_v(pstbase_screen_uart, 0, "hello x=%d y=%d\r\n", x, y);
    }


================================================================================
三、推荐写法（用分级宏，自动加前缀）
================================================================================

  uart_log.h 提供了四个宏，自动在前面拼 [DBG]/[INF]/[WRN]/[ERR]：

    uart_dbg(uart, fmt, ...)   → [DBG] ...
    uart_info(uart, fmt, ...)  → [INF] ...
    uart_warn(uart, fmt, ...)  → [WRN] ...
    uart_err(uart, fmt, ...)   → [ERR] ...

  例子：

    #include "uarts.h"
    #include "uart_log.h"

    void my_print_demo(void)
    {
        uint16_t bat = 1250;                        /* 12.50V 的×100表示 */

        uart_info(pstbase_screen_uart, "bat: %u.%02uV\r\n", bat/100, bat%100);
        uart_warn(pstbase_screen_uart, "queue full\r\n");
        uart_err(pstbase_screen_uart, "crc failed, cmd=0x%02x\r\n", 0xA5);
    }

  注意：level 参数目前没做过滤（uart_log.c 里 (void)level），传宏内部的
        0/1/2/3 都会原样发出去。宏只是帮你加前缀文本，不裁剪日志。


================================================================================
四、在任务里周期打印（最常见用法）
================================================================================

  打印遥测/调试数据，典型就是放任务循环里，每 N ms 发一次：

    void my_task(void *argument)
    {
        for (;;) {
            int v = read_sensor();
            uart_info(pstbase_screen_uart, "v=%d\r\n", v);
            osDelay(100);                /* 10Hz 够了，别太高 */
        }
    }

  照 mission_planner.c 现有写法抄就行，例如它里面就是：
    uart_printf_v(pstbase_screen_uart, 0, "[MSN] zones: %d\r\n", count);


================================================================================
五、新手易卡点
================================================================================

【1. 第一参必须是 pstbase_screen_uart，别传错】
  这是 uarts.h 里 extern 的全局指针，指向 USART2 的基类。
  传成 pstbase_ground_uart 就发到飞控那路了（USART1），数据不对路。

【2. 要自己加 \r\n】
  uart_printf_v 不自动加换行。忘了加，串口助手看到的就是一长串连在一起。

【3. 别在 HAL 回调/中断里直接发】
  收数据可能在任务或中断上下文里。发屏要走 TX 队列（uart_transmit 非阻塞）。
  正确做法：任务循环里取数据 → uart_printf_v 发。别在 HAL_UART_RxCplt
  这类回调里直接调，会和正常收发竞争。

【4. 单条别超 127 字节】
  buf 是 128B 含 '\0'。一条想打很长的表，会被截断。拆成几条发。

【5. 必须在驱动初始化之后才能用】
  pstbase_screen_uart 是在 uart_board_init()（DRIVER_INIT_1 注册）里才建好的。
  所以不能在 main() 一进去、或 HAL_UART_MspInit 这种早期阶段就用——那时
  指针还没赋值（NULL），uart_printf_v 第一行 NULL 检查会直接返回 -1。
  在 FreeRTOS 任务里、或 main 里 MX_xxx_Init() 都跑完之后用，都没问题。

【6. 波特率 9600，别期望太高吞吐】
  USART2 = 9600，约 960 字节/秒。一堆 debug 信息猛发会堵，丢帧不报错。
  调试够用，别拿来刷高频波形数据。要高吞吐走 USART1/USART3（500000）。

六、对照速查
================================================================================

  想做的事                          怎么写
  ─────────────────────────────── ────────────────────────────────────────
  打一行普通文本到串口2            uart_printf_v(pstbase_screen_uart,0,"hi\r\n");
  打带变量的调试信息               uart_dbg(pstbase_screen_uart,"x=%d\r\n",x);
  打带级别的日志                   uart_info / uart_warn / uart_err（同上签名）
  周期刷新某数据                   任务循环里 osDelay(N) + 上面任一宏
  打 16 进制                       uart_printf_v(...,"cmd=0x%02X\r\n",byte);
  打浮点（项目多用×100整数存）     uart_info(...,"v=%d.%02d\r\n",v/100,v%100);

  头文件：#include "uarts.h" + #include "uart_log.h"
  串口2 句柄：pstbase_screen_uart
  波特率：9600（usart.c 里 MX_USART2_UART_Init 定的，要改在那改）


================================================================================
七、和《陶晶驰屏数据展示工作指南.txt》的区别
================================================================================

  那份讲的是"把数据格式化成屏控件指令（t0.txt=...\xff）发给屏显示"，
  关心的是屏端 UI 控件名和 0xFF 结尾。

  本份讲的是"纯粹打印文本看数据"（调试/日志用），用 \r\n 结尾，给串口
  助手看。两者底层都走 uart_printf_v + pstbase_screen_uart，但格式串和
  目的不同：
    看数据调试 → 本份，\r\n 结尾
    喂屏控件   → 那份，0xFF 结尾 + 控件名

  别混：调试用 \r\n，喂屏用 0xFF。
